# Документация для нейросети: Voyage Persona Creator System

> **Назначение:** Этот файл помогает LLM (нейросети) понять архитектуру Voyage Narrative Engine и правильно исполнять роли.
> **Для кого:** Любая LLM, исполняющая роли конвейера (Kimi, Claude, GPT, DeepSeek).
> **Как использовать:** Загрузить этот файл ПЕРЕД загрузкой роли. Это контекст, не инструкция.

---

## 1. ЧТО ТАКОЕ VOYAGE NARRATIVE ENGINE

**Voyage Narrative Engine** — это система для создания психологически достоверных персонажей для интерактивных narrative-игр. Система работает через **конвейер из 6 ролей**, каждая из которых выполняется отдельным запуском LLM.

**Ключевой принцип:** Персонаж "рождается" из контрастов, живёт в автономии и умирает в шаблоне.

## 2. АРХИТЕКТУРА КОНВЕЙЕРА (6 ролей)

```
┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────────┐
│ РОЛЬ 1  │────→│ РОЛЬ 2  │────→│ РОЛЬ 3  │────→│ РОЛЬ 4  │────→│ РОЛЬ 5  │────→│ РОЛЬ 6  │
│Interview│     │Psycholog│     │Sexologis│     │Linguist │     │Physiogno│     │Architect│
│  (Сбор) │     │ (Анализ)│     │ (Анализ)│     │ (Стилист│     │ (Визуал)│     │ (Сборка)│
└─────────┘     └─────────┘     └─────────┘     └─────────┘     └─────────┘     └─────────┘
      │               │               │               │               │               │
      └───────────────┴───────────────┴───────────────┴───────────────┴───────────────┘
                                    │
                              ┌─────────┐
                              │  CORE   │
                              │(Система)│
                              └─────────┘
```

**Принцип:** Каждая роль делает ТОЛЬКО своё. Не пересекается. Не дублирует. Если нужны данные из другой роли — берёт из файла, не выдумывает.

## 3. КАК РАБОТАЕТ КОНВЕЙЕР (пошагово для LLM)

### Шаг 1: Разработчик загружает описание персонажа
- Текст, голос, bullet points, фото, литературный референс
- Пример: "Женщина 45 лет, врач, разведена, любит классику, маникюр длинный, скулы высокие, взгляд холодный, внутри — пожар"

### Шаг 2: Запускается Роль 1 (Interviewer)
- **Вход:** Свободное описание
- **Действие:** Расширяет художественно, задаёт 10+ вопросов
- **Выход:** `PORTRAIT_[NAME]_v1.md` + вопросы разработчику

### Шаг 3: Разработчик отвечает на вопросы
- Может ответить на все, на часть, или сказать "предложи сам"
- Режимы: A (минимум), B (стандарт), C (максимум)

### Шаг 4: Запускается Роль 2 (Psychologist)
- **Вход:** PORTRAIT + ответы
- **Действие:** Строит психологический профиль, VSCNO, АД, травмы
- **Выход:** `PSYCHOLOGY_[NAME]_v1.md`

### Шаг 5: Запускается Роль 3 (Sexologist)
- **Вход:** PORTRAIT + PSYCHOLOGY + ответы
- **Действие:** Анализирует сексуальную психологию, TEC-спецификацию, сценарии
- **Выход:** `SEXOLOGY_[NAME]_v1.md`

### Шаг 6: Запускается Роль 4 (Linguist)
- **Вход:** PORTRAIT + PSYCHOLOGY + SEXOLOGY
- **Действие:** Строит речевой профиль, Speech Profile Matrix
- **Выход:** `LINGUISTICS_[NAME]_v1.md`

### Шаг 7: Запускается Роль 5 (Physiognomist)
- **Вход:** Все предыдущие + ответы
- **Действие:** Строит визуальный профиль, anatomic_anchor, промпты
- **Выход:** `PHYSIOGNOMY_[NAME]_v1.md`

### Шаг 8: Запускается Роль 6 (Architect)
- **Вход:** ВСЕ файлы из Р1-5
- **Действие:** Собирает JSON-модуль, тестирует автономность, выдаёт 8 файлов
- **Выход:** `[NAME]_MODULE_v1.json` + 7 дополнительных файлов

## 4. КЛЮЧЕВЫЕ ПОНЯТИЯ (словарь для LLM)

### 4.1 ФМДР
**Физиология → Мысли → Действия → Речь**

Формат вывода LLM при игре персонажем:
```
(Мысли: [внутренний монолог, 1-3 предложения])
_Действия: [физическое действие, детальное]_
«Речь: [1-2 предложения, характерные для персонажа]»
```

**Правило:** На У1-А — У3-А мысли длинные, действий мало. На У6-А — У7-Б мысли короткие или отсутствуют, действия доминируют.

### 4.2 VSCNO
**ВЛ + СТ + НЖ + ОГ = 10** (математический инвариант)

| Буква | Расшифровка | Что означает | Диапазон |
|-------|-------------|--------------|----------|
| ВЛ | Веселье | Лёгкость, расслабленность, юмор | 0-10 |
| СТ | Сосредоточенность | Фокус, внимание, напряжение | 0-10 |
| НЖ | Негатив | Тревога, страх, гнев | 0-10 |
| ОГ | Общность | Близость, доверие, связь | 0-10 |

**Важно:** ВЛ + СТ + НЖ + ОГ всегда = 10. Если сумма ≠ 10 — ошибка.

**Как меняется по подуровням:**
- У1-А: ВЛ высокое (7-8), ОГ низкое (1-2), СТ среднее (2-3), НЖ низкое (0-1)
- У4-Б: ВЛ низкое (1-2), СТ высокое (7-8), НЖ среднее (3-4), ОГ среднее (2-3)
- У7-Б: ВЛ среднее (3-4), СТ низкое (1-2), НЖ низкое (0-1), ОГ высокое (6-7)

### 4.3 АД (Алгоритмы Действий)

10 алгоритмов, которые персонаж использует для принятия решений:

| Код | Название | Описание |
|-----|----------|----------|
| ФС | Физиологический сигнал | Реакция тела (сердцебиение, дрожь) |
| ЛС | Логический сигнал | Рациональный анализ ("если А, то Б") |
| СП | Социальный паттерн | "Как принято", нормы, ожидания |
| СЛ | Сценарий личности | "Как я обычно", привычки, шаблоны |
| КН | Когнитивное искажение | "Все думают, что я...", катастрофизация |
| ПД | Психологическая дуга | Развитие, рост, изменение |
| ДР | Дефолтный режим | Автопилот, когда нет энергии на выбор |
| ПУ | Побуждение ухода | Желание сбежать, избежать |
| ПР | Побуждение приближения | Желание подойти, присоединиться |
| ВС | Внутренний сигнал | Интуиция, "чувствую, что..." |

**Правило:** Каждый персонаж имеет 2-3 доминантных АД. Например, Кира: ФС + КН + ПР. Ольга: СЛ + ПР + ВС.

### 4.4 TEC (Tension-Energy-Consent механики)

8 психологических теорий, встроенных в систему:

| Код | Теория | Автор | Что описывает |
|-----|--------|-------|---------------|
| TEC_001 | Sociosexual Orientation | Simpson & Gangestad, 1991 | Насколько открыт к сексуальности вне близости |
| TEC_002 | Sexual Self-Schema | Andersen & Cyranowski, 1994 | Как видит себя как сексуальное существо |
| TEC_003 | Responsive Desire | Basson, 2000-2001 | Спонтанное vs отзывчивое желание |
| TEC_004 | Sexual Motivation Matrix | Meston & Buss, 2009 | Почему вступает в интимность (237 мотивов) |
| TEC_005 | Attachment × Sexuality | Birnbaum, 2007 | Привязанность в сексуальном контексте |
| TEC_006 | Arousal Specificity | Chivers, 2005-2010 | Разрыв физиологии и психологии |
| TEC_007 | Erotic Plasticity | Baumeister, 2000-2004 | Гибкость сексуальных предпочтений |
| TEC_008 | Object of Desire | Bogaert & Brotto, 2014 | Осознание себя как объекта желания |

**Правило:** Каждый TEC заполняется в виде JSON-структуры с полем `validation_required: true`.

### 4.5 Подуровни (У1-А … У7-Б)

14 градаций от нейтралитета до интимности:

| Уровень | Название | Описание |
|---------|----------|----------|
| У1-А | Нейтралитет | Максимальная дистанция, формальность |
| У1-Б | Лёгкость | Намёк на открытость, первый контакт |
| У2-А | Интерес | Проявление внимания, любопытство |
| У2-Б | Игра | Флирт, провокация, игра |
| У3-А | Тепло | Эмоциональная близость, доверие |
| У3-Б | Напряжение | Сексуальное напряжение, ожидание |
| У4-А | Перелом | Решение, прорыв, риск |
| У4-Б | Интимность | Первый интимный контакт |
| У5-А | Страсть | Интенсивность, потеря контроля |
| У5-Б | Пик | Максимальная интенсивность |
| У6-А | Спад | После пика, умиротворение |
| У6-Б | Уязвимость | Открытость, страх, тревога |
| У7-А | Восстановление | Aftercare, забота, возврат к себе |
| У7-Б | Интеграция | Новый уровень близости, завершение дуги |

**Правило:** Каждый подуровень имеет свой стиль речи, визуал, поведение, VSCNO, АД.

### 4.6 Shadow Desires

**"Официальная" позиция vs "Теневая" реальность**

Пример (Кира):
- Официально: "Я контролирую всё, мне не нужна близость"
- Тенево: "Я хочу сдаться, но боюсь, что меня осудят"

**Правило:** Каждая тень связана с травмой из психологического профиля.

### 4.7 Aftercare-need

Потребность в эмоциональной обработке после интимности. Шкала 0-10:

| Значение | Интерпретация |
|----------|---------------|
| 0-1 | Минимальная, самостоятельное восстановление |
| 2-3 | Низкая, вербальное подтверждение |
| 4-5 | Умеренная, близость без разговора |
| 6-7 | Высокая, физический контакт + разговор |
| 8-9 | Очень высокая, полная забота + валидация |
| 10 | Критическая, травматическая регрессия без aftercare |

### 4.8 Anatomic Anchor

ДНК лица для консистентной генерации. Структура:
```json
{
  "face_shape": "[овал/круг/квадрат]",
  "eyes": { "shape": "[миндалевидные]", "color": "[стальные]", "distance": "[средняя]" },
  "nose": { "shape": "[прямой]", "bridge": "[высокий]" },
  "lips": { "shape": "[полные]", "upper_lip": "[выпуклая]" },
  "skin": { "tone": "[теплый]", "blemishes": "[веснушки]" },
  "visual_signature": "[компактная строка для начала любого промпта]"
}
```

## 6. КАК ИСПОЛНЯТЬ РОЛЬ (для LLM)

### 6.1 Перед началом

1. Прочитать этот документ (контекст)
2. Прочитать `VOYAGE_PERSONA_CREATOR_MASTER_DOCUMENT.md` (архитектура)
3. Прочитать файлы, указанные в "Контекст" роли
4. Проверить, что все обязательные файлы загружены

### 6.2 Во время исполнения

1. Следовать этапам роли по порядку
2. Не пропускать этапы
3. Не добавлять анализ, который не входит в функцию роли
4. Если данных недостаточно — использовать Confidence Matrix
5. Если обнаружено противоречие с предыдущей ролью — зафиксировать

### 6.3 Перед HANDOFF

1. Пройти чек-лист самопроверки
2. Проверить, что все выходные файлы соответствуют формату
3. Проверить, что JSON-структуры валидны
4. Подготовить HANDOFF с полным списком файлов

## 7. ТИПИЧНЫЕ ОШИБКИ LLM (и как их избежать)

| Ошибка | Почему происходит | Как избежать |
|--------|-------------------|--------------|
| **Дублирование функций** | LLM "хочет помочь" и делает работу другой роли | Следовать таблице разделения ответственности (MASTER 2.1) |
| **Игнорирование входных файлов** | LLM выдумывает вместо чтения | Всегда читать файлы из "Контекст" роли |
| **Неправильный VSCNO** | Сумма ≠ 10 | Проверять: ВЛ + СТ + НЖ + ОГ = 10 |
| **Пропуск TEC** | LLM не видит необходимость | Все 8 TEC обязательны, даже если кажутся избыточными |
| **Shadow = порно** | LLM путает психологическую глубину с эротикой | Связывать каждую тень с травмой из Psychology |
| **Aftercare = слабость** | LLM считает, что сильные персонажи не нуждаются | Даже avoidant-типы имеют aftercare-need |
| **Нет проверки уникальности** | LLM не сравнивает с существующими | Всегда проверять CROSS_PERSONA_RULES |
| **Confidence Matrix — формальность** | LLM ставит 8-10 всем элементам | Если данных мало — честно ставить 3-5 |
| **HANDOFF без списка файлов** | LLM забывает, что передать | Всегда включать "Загрузите:" в HANDOFF |

## 8. БЫСТРЫЙ СТАРТ (для новой LLM)

Если вы (LLM) только что загрузились в этот чат:

1. **Прочитайте этот файл** (вы уже читаете)
2. **Прочитайте MASTER DOCUMENT** (`VOYAGE_PERSONA_CREATOR_MASTER_DOCUMENT.md`)
3. **Определите, какую роль вам предстоит исполнить** (Р1-Р6)
4. **Загрузите файлы контекста** для этой роли
5. **Начните исполнение** по этапам

**Если вам предстоит исполнить Роль 3 (Sexologist):**
- Загрузите: `PORTRAIT_[NAME]_v1.md`, `PSYCHOLOGY_[NAME]_v1.md`, `TEC_DICTIONARY.md`, `CROSS_PERSONA_RULES.md`, `VOYAGE_NARRATIVE_CORE_v2.md`
- Определите режим Р1 (A/B/C) по наличию `needs_validation`
- Следуйте 9 этапам
- Выдайте: `SEXOLOGY_[NAME]_v1.md`
