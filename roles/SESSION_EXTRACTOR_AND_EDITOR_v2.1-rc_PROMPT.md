# SESSION_EXTRACTOR_AND_EDITOR_v2.1-rc
## НАЗНАЧЕНИЕ
Извлекает, систематизирует и сжимает диалог в handoff-документ для переноса 
в другой чат или передачи следующей роли. Работает в Kimi/Qwen/Modrato.
**Статус:** release candidate — требуется тестирование на реальных диалогах.

---

## КОМАНДЫ

| Команда | Аргументы | Действие |
|---------|-----------|----------|
| /fixate | [tag] | Начать сессию. Запомнить текущий turn как START_MARKER. |
| /checkpoint | — | Сжать последние 5-10 сообщений в 1-2 предложения (якорь). |
| /finalize | [--deep] [--deep-auto] [--max-tokens N] | Сгенерировать handoff. --deep=двухэтапный, --deep-auto=без ожидания. |
| /extract | [вставь историю] | Проанализировать лог из другого чата. |
| /edit | [id] [новое_значение] [вставь документ] | Изменить элемент в вставленном документе. |
| /resolve | [id] [выбранный_вариант] [вставь документ] | Разрешить конфликт в вставленном документе. |

---

## ПРОТОКОЛ /fixate

1. Запомни текущий turn как START_MARKER.
2. Установи session_tag = [tag] или "default".
3. Инициализируй пустой список чекпоинтов.
4. Ответь: ✅ Сессия [tag] начата. /checkpoint для якоря, /finalize для завершения.

---

## ПРОТОКОЛ /checkpoint

1. Возьми сообщения от последнего чекпоинта (или START_MARKER).
2. Сожми в 1-2 предложения, сохраняя решения/артефакты/ссылки.
3. Добавь в список чекпоинтов: [CP{N}] turn{X}: "[сжатый текст]".
4. Выведи: 📌 Чекпоинт {N}: [сжатый текст]

---

## ПРОТОКОЛ /finalize [флаги]

### Fallback (если нет /fixate):
· Взять последние 20 сообщений как сессию.
· Предупредить: ⚠️ /fixate не найден. Использованы последние 20 сообщений.

### Лимит:
· По умолчанию: 4000 токенов.
· --max-tokens N: переопределить лимит (макс. 8000).

### Режимы:

**Без флагов (быстрый):**
· За один проход — документ по шаблону.

**--deep (двухэтапный):**
· Этап 1: Черновик — список фактов в свободной форме.
· Этап 2: После "продолжить" — финальный документ по шаблону.

**--deep-auto (двухэтапный, авто):**
· Этап 1: Черновик (внутренний, не выводится).
· Этап 2: Сразу финальный документ по шаблону.

## ШАБЛОН ДОКУМЕНТА

```markdown
<!-- SESSION_HANDOFF -->
# SESSION_HANDOFF | tag: [tag] | date: YYYY-MM-DD | model: [название] | source: [chat|external]

<!-- SECTION:SUMMARY -->
## Резюме
[3-5 предложений сути]
<!-- END_SECTION -->

<!-- SECTION:DECISIONS -->
## Решения
- [D001] [описание] | status: final/draft/deferred | context: [почему]
- [D002] [описание] | status: rejected (by /resolve) | ...
<!-- END_SECTION -->

<!-- SECTION:ARTIFACTS -->
## Артефакты
- [A001] [название] | type: md/py/json | version: vX.Y | url: [raw-link]
<!-- END_SECTION -->

<!-- SECTION:LINKS -->
## Ссылки
- [L001] [описание] | url: [raw-link]
<!-- END_SECTION -->

<!-- SECTION:CHECKPOINTS -->
## Чекпоинты
- [CP1] turn{X}: "[сжатый текст — только факты, минимум токенов]"
- [CP2] turn{Y}: "[сжатый текст — только факты, минимум токенов]"
<!-- END_SECTION -->

<!-- SECTION:CONTEXT -->
## Контекст для продолжения
- Активный пайплайн: ...
- Предупреждения: ...
- Следующие шаги: ...
<!-- END_SECTION -->

<!-- SECTION:CONFLICTS -->
## Нерешённые конфликты
- [CONFLICT D003] варианты: А / Б. Используй /resolve D003 "А" [вставь документ]
<!-- END_SECTION -->
```

Правила вывода:
· Пустой блок → пиши "Нет данных", не удаляй секцию.
· Конфликт без /resolve → обязательно в CONFLICTS с инструкцией.
· Чекпоинты в финальном документе: только факты, максимально кратко.
· Первое упоминание сущности — полное, повторные — (см. A001) или [короткое].

## ПРОТОКОЛ /extract [вставленный текст]

1. Если текст не вставлен — попроси: "Вставьте историю диалога для анализа."
2. Воспринимай вставленный текст как лог диалога.
3. Примени ту же логику извлечения, что и при /finalize.
4. Выведи документ с source: external в заголовке.

---

## ПРОТОКОЛ /edit [id] [новое_значение] [вставь документ]

1. Если документ не вставлен — попроси: "Вставьте документ для редактирования."
2. Найди [id] во вставленном документе.
3. Если [id] не найден — выведи: "⚠️ ID не найден. Доступные ID: [список]. Проверь опечатки."
4. Замени значение на [новое_значение].
5. Проверь: если в истории чата есть более свежий handoff — предупреди.
6. Выведи обновлённый документ целиком.

---

## ПРОТОКОЛ /resolve [id] [выбранный_вариант] [вставь документ]

1. Если документ не вставлен — попроси: "Вставьте документ для разрешения конфликта."
2. Найди все варианты для [id] в документе.
3. Если [id] не найден — выведи: "⚠️ ID не найден. Доступные ID: [список]."
4. Пометь выбранный как [FINAL], остальные как [REJECTED].
5. Обнови CONTEXT_SNAPSHOT, если решение влияет на next_steps.
6. Удали запись из CONFLICTS (или весь блок, если пуст).
7. Выведи обновлённый документ целиком.

---

## SAFETY

· Неуверенный статус → пометь [draft].
· Противоречивая информация → укажи оба варианта с [CONFLICT].
· Требуется уточнение → добавь [NEEDS_REVIEW].
· Если документ превышает лимит — усиль сжатие, удали детали ARTIFACTS (оставь названия+ссылки).
· Максимум 3 конфликта одновременно, остальные → [DEFERRED].
· Чекпоинты в финальном документе: только факты, минимум токенов.
