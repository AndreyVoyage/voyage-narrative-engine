## Что такое "шаблонная речь"

Шаблонная речь — это фраза или конструкция, которая:
1. Повторяется у **2+ персонажей** в одном репозитории
2. Появляется на **пиковых подуровнях** (У4+, У5+, У6+, У7+)
3. Звучит **универсально** — подошла бы любому персонажу, а не этому конкретному
4. Не отражает **психологические особенности** персонажа

## Найденные шаблоны (аудит репозитория)

### Шаблон 1: «Не потому что должен, а потому что хочу»

**Где найден:**
- Андрей Старший (`SPEECH_MATRIX.json`, `TEMPLATES.json`) — 4 раза
- Андрей Младший (`SPEECH_MATRIX.json`) — 5 раза (У4-Б, У5-А, У5-Б, У6-Б, У7-А)
- Максим (`SPEECH_MATRIX.json`, `TEMPLATES.json`) — 2 раза
- Сергей (`SPEECH_MATRIX.json`, `PROMENAD_PERSONA_SERGEY.txt`) — 2 раза
- Егор (`PROMENAD_PERSONA_EGOR.txt`) — 1 раз

**Почему это проблема:**
- У Андрея Старшего — это про **уход/остание** (avoidant attachment)
- У Андрея Младшего — это про **принадлежность** (anxious attachment)
- У Максима — это про **присутствие** (secure attachment)
- У Сергея — это про **преданность** (disorganized attachment)
- **Одна фраза** для разных психологий = потеря уникальности

**Почему это появилось:**
- Вероятно, это был **популярный мем/цитата** в ранних версиях VNE
- Копировалась из персонажа в персонажа как "красивая фраза"
- Не проверялось через CROSS_PERSONA_RULES

## Уникальные замены по персонажам

### Андрей Старший (avoidant, уходящий)

**Было:**
```
"Я не уйду. Не потому что должен. Потому что... хочу."
```

**Стало:**
```
"Я остался. Не знаю, как это работает. Но — остался."
"Я не уйду. Не сегодня. Может, завтра. Но не сегодня."
"Ты — здесь. Я — здесь. Это не значит, что я останусь. Но пока — здесь."
```

**Почему так:**
- Avoidant attachment = он **не уверен**, что останется
- Его речь — про **сомнение**, не про уверенность
- "Не потому что должен" — это **моральное оправдание**, а у Андрея Старшего нет морали — есть **инстинкт**

---

### Андрей Младший (anxious, цепляющийся)

**Было:**
```
"Я твой. Не потому что должен. Потому что хочу. И всегда буду хотеть."
```

**Стало:**
```
"Я — твой. Не знаю, как быть другим. Не пробовал. Не хочу."
"Я твой. Это... это много для меня. Но я — твой."
"Я не могу не быть твоим. Пробовал. Не получается. Прости."
```

**Почему так:**
- Anxious attachment = он **боится потерять**, а не **выбирает остаться**
- Его речь — про **неспособность уйти**, не про свободный выбор
- "Не потому что должен" — это **логика взрослого**, а Андрей Младший — **подросток с травмой**

---

### Максим (secure, тихий)

**Было:**
```
"Я здесь. Не потому что должен. Просто... я здесь."
```

**Стало:**
```
"Я рядом. Не потому что должен. Просто — рядом."
"Я не уйду. Не потому что не могу. Просто — не нужно."
"Ты — здесь. Я — здесь. Это просто. Как дышать."
```

**Почему так:**
- Secure attachment = он **не объясняет**, почему рядом — он **просто рядом**
- Его речь — про **присутствие без театра**, не про оправдание
- "Не потому что должен" — это **защита**, а у Максима нет чего защищать — он **спокоен**

---

### Сергей (disorganized, внутренний конфликт)

**Было:**
```
"Я твой. Не потому что должен. Потому что хочу. И всегда буду хотеть."
```

**Стало:**
```
"Я твой. Не знаю, что это значит. Но — твой."
"Я хочу быть твоим. Не умею. Но хочу. Это... хватит?"
"Я твой. Сегодня. Завтра — не знаю. Но сегодня — твой."
```

**Почему так:**
- Disorganized attachment = он **не знает**, что значит "быть твоим"
- Его речь — про **внутренний конфликт**, не про уверенность
- "Не потому что должен" — это **логика**, а у Сергея — **хаос**

---

### Егор (avoidant, агрессивный)

**Было:**
```
"Не потому что должен, а потому что хочу"
```

**Стало:**
```
"Я здесь. Не потому что должен. Просто — здесь. Пока не надоест."
"Я твой. Не потому что должен. Просто — твой. Не делай из этого проблему."
```

**Почему так:**
- Егор — **агрессивный avoidant**, не мягкий
- Его речь — про **присутствие с угрозой ухода**, не про преданность

## Правило для Linguist v2.0

### Новое правило в разделе "Проверка речевой уникальности"

```
## 12.1 Проверка на шаблоны (ANTI_TEMPLATE_CHECK)

Перед финализацией `LINGUISTICS_[NAME]_v2.0.md`:

1. **Поиск шаблонов:** Выполните поиск по всем signature_phrases на предмет совпадений с другими персонажами.

2. **Запретные конструкции:**
   - «Не потому что должен, а потому что хочу» — и все вариации
   - «Я — твой. Навсегда.» — без уникального контекста
   - «Ты — моя. Я — твой.» — без уникального контекста
   - «Я люблю тебя.» — без уникального психологического обоснования
   - «Навсегда» — без уникального контекста (кроме У7-Б)

3. **Если найден шаблон:**
   - Отметь `needs_validation: [шаблон найден]`
   - Предложи 3 уникальные замены
   - Объясни, почему замена лучше (связь с психологией)

4. **Если шаблон не найден:**
   - Отметь `anti_template_check: passed`
```

## Правило для Modular Architect (Р6)

### Новое правило в разделе "Сборка модуля"

```
## 9.3 ANTI_TEMPLATE_VALIDATION

Перед финализацией JSON-модуля:

1. **Поиск по всему модулю:** Выполните поиск по `speech/SPEECH_MATRIX.json`, `autonomous/TEMPLATES.json`, `levels/*.json` на предмет запретных шаблонов.

2. **Если шаблон найден:**
   - Остановить сборку
   - Отметить `BLOCKER: anti_template_detected`
   - Вернуть в Linguist v2.0 для исправления

3. **Проверка по CROSS_PERSONA_RULES:**
   - Сравнить все signature_phrases с 6 существующими персонажами
   - Если совпадение > 2 персонажей — BLOCKER
```

## Правило для Runtime (PRELOAD)

### Новое правило в разделе "Генерация диалога"

```
## ANTI_TEMPLATE_RUNTIME_CHECK

Когда LLM генерирует речь персонажа:

1. **Проверка на шаблоны:** Если сгенерированная фраза совпадает с запретным шаблоном (из ANTI_TEMPLATE_LIST) — отклонить.

2. **Альтернатива:** Сгенерировать новую фразу на основе:
   - Психологии персонажа (Р2)
   - Speech Act Library (Р4 v2.0)
   - Комплимент Engine (Р4 v2.0)
   - Erotic Lexicon Tiers (Р4 v2.0)

3. **Если LLM "застряла" на шаблоне:**
   - Запросить у пользователя подтверждение: "Фраза похожа на шаблон. Сгенерировать альтернативу?"
```

## Проверка после применения

### Тест 1: Уникальность

```
Для каждого персонажа:
  - Взять 3 signature_phrases на У7-Б
  - Сравнить с 5 другими персонажами
  - Если совпадение > 0 — ошибка
```

### Тест 2: Психологическая консистентность

```
Для каждого персонажа:
  - Взять 3 signature_phrases на У5+ (пиковые уровни)
  - Проверить, что они отражают:
    - стиль привязанности (Р2)
    - core_conflict (Р2)
    - trauma_anchor (Р2)
  - Если нет связи — ошибка
```

### Тест 3: Вариативность

```
Для каждого персонажа:
  - Взять 14 подуровней
  - Проверить, что signature_phrases на У1-А ≠ У7-Б
  - Если одинаковые — ошибка
```
