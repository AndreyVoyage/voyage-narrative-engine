## 0. ЧТО МЫ СОЗДАЁМ (Vision в 1 предложении)

**Voyage Narrative Engine** — это AI-Native интерактивная система, где пользователь ведёт эмерджентные диалоги с проработанными психологическими персонажами (Кира, Сергей, Марина, Максим) в сценариях (сауна, прогулка, знакомство), а система управляет памятью персонажей, их эмоциональным состоянием и генерирует визуальные промпты.

### 🔧 Что мы сейчас делаем (Фаза 1 — Foundation)

**Создаём единый стандарт данных.**

Все персонажи сейчас хранятся в `profile.md` (проза) и разрозненных `memory.json`. Это не масштабируется: нельзя автоматически валидировать, нельзя программно читать состояние, нельзя тестировать.

**Текущая задача:** Создать `persona_schema_v3_1_VOYAGE.json` → написать `migrate_personas.py` → конвертировать всех 4 персонажей в валидные JSON-модули → проверить тестами.

Это **фундамент**. Без него нельзя строить Orchestrator, потому что Orchestrator должен читать структурированные данные, а не парсить Markdown.

## 4. КАК МЫ РАБОТАЕМ (Workflow)

### В этом чате (Kimi)
- **Архитектура, планирование, анализ** — то, что требует длинного контекста (200K токенов)
- **JSON Schema, миграторы, скрипты** — код, который нужно сохранить в файлы
- **Рефлексия** — анализ ответов других AI, проверка логики

### В других чатах (DeepSeek / Qwen)
- **Психологическая глубина** — TEC-конструкты, научные обоснования
- **Тестовые сценарии** — прогон тестов CHIVERS/BAUMEISTER в диалоговом режиме
- **Генерация сценариев** — creative writing, если нужен новый сценарий

### Правило переключения
> **Если задача требует «сохранить файл в репозиторий» → Kimi.**
> **Если задача требует «проверить, нарушает ли Кира профиль в диалоге» → DeepSeek/Qwen.**

### Git-дискиплина
1. Каждая завершённая фаза = commit
2. Каждый файл из этого чата = скачать → положить в репозиторий → `git add . && git commit -m "phase-X: description"`
3. После каждой сессии: обновить `memory.json` → commit
4. Не делать `git push` без проверки (чтобы не залить лишние `.bak`)

## 5. РИСКИ И ЗАПАСНЫЕ ПЛАНЫ

| Риск | Вероятность | Запасной план |
|------|-------------|---------------|
| Локальные LLM (Ollama) слишком слабые для психологической глубины | Средняя | Оставить Kimi/DeepSeek как основной движок, Ollama — только для оффлайн-заглушки |
| Termux не тянет Stable Diffusion | Высокая | Генерация картинок только через API (Kimi Image / Midjourney), в Termux — текстовый режим |
| JSON Schema тормозит разработку | Низкая | Упростить Schema до «required только id + name + internal_state», остальное — free-form |
| Персонажи «ломаются» в E2E тестах | Средняя | Вернуться к Фазе 2, усилить system prompt, добавить few-shot examples |
