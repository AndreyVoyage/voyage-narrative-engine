> **Narrative baseline:** `main == 435e1cf` (β-трек зелёный).

## 0. Что это и главный принцип

Игрок в игре выбирает «Поговорить наедине» с персонажем → сцена ставится на паузу → открывается приватный LLM-чат. Персонаж помнит канон **до текущего момента** и **прошлые приватные беседы**, но **основную игру не меняет**. По выходе игра продолжается без изменений.

**Главный инвариант:** `canon НЕ читает aside; aside читает canon; aside НЕ пишет в canon.` Односторонняя связь.

---

## 1. Источник снимка канона (ключевое открытие)

Генерённый `novel/game/scenes_v2_generated.rpy` **уже трекает канон-состояние в RenPy-переменных**:
`v2_flags` (set), `v2_completed_scenes` (set), `v2_levels` (dict), `v2_relationships` (dict), и применяет effects по ходу игры.

Поэтому Character Aside **НЕ требует** сначала строить полный live JSON-runtime (N5G). Снимок канона берётся **прямо из этих RenPy-переменных** + текущая сцена/beat:

```json
"canon_snapshot": {
  "scene_id": "SC_017", "beat_id": "sc_017_v2_1a", "progress_index": <int>,
  "flags": [ ...из v2_flags... ],
  "completed_scenes": [ ...из v2_completed_scenes... ],
  "levels": { ...v2_levels... },
  "relationships": { ...v2_relationships... }
}
```

**Past-only:** снимок содержит только уже произошедшее. Будущие beats/сцены/невыставленные флаги в контекст aside НЕ подаются, даже если известны рендереру.

---

## 2. Сборка персоны

Персонаж для aside собирается из **модульного источника** `personas/<id>/` (источник правды по `NARRATIVE_DECISIONS_v1 §2`), компактно, переиспользуя существующий `scripts/python/build_prompt_modular.py`.

- Берём: core/identity, психология, речь, отношения (к POV/другим), текущий уровень (из `levels[<id>]`).
- Компактно, не весь модуль целиком (управление длиной контекста).
- Выбор персонажа поэтапно: сначала уже введённые/доступные по снимку; позже — introduced-but-absent.

---

## 3. Модель памяти (Persistent, non-canonical)

Две параллельные памяти на персонажа:

```text
1. Канон-таймлайн (RenPy v2_* + player_state) — ведёт игру.
2. Aside-таймлайн (приватная память) — копит прошлые беседы; НЕ сливается в канон.
```

**Хранение (отдельно от канона):**
```text
private_chats/<save_slot>/<character_id>/
    memory_summary.json           # суммаризованная сводка прошлых бесед
    sessions/<scene>_<beat>_NNN.json
```

**Защита от «памяти из будущего».** Aside-память привязана к `save_slot` + `progress_index`. Правило: персонаж помнит только те беседы, что произошли **не позже текущей канон-точки**. Откат save назад → память из будущего в контекст НЕ подаётся.

**Три режима памяти (настройка):** `OFF` (забывается после выхода) / `SESSION` (в пределах запуска) / `PERSISTENT` (в рамках save slot, per-character) — рекомендуемый `PERSISTENT`.

**Рост контекста → суммаризация:** в контекст идёт `memory_summary` + последние 10–20 сообщений текущей сессии, а не весь transcript.

## 4. LLMProvider (абстракция)

```text
LLMProvider.complete(messages, params) → reply
Реализации: mock | local(Ollama) | cloud
```

- **mock** — детерминированный, для тестов и первого играбельного прототипа (ноль стоимости/сети).
- **local (Ollama)** — предпочтителен для взрослого контента (приватность, офлайн, без контент-политик; нужен GPU).
- **cloud** — опционально; помнить про контент-политики и стоимость.
- Смена провайдера **не меняет** остальной код (Aside зависит от интерфейса, не от сервиса).

---

## 5. Сборка промпта (recipe)

```text
1) system: персона (§2) + safety/tone (RN-SAFETY-STYLE-1, content_rating) + рамка «здесь, наедине»
2) canon snapshot (past-only, §1)
3) aside memory summary (§3)
4) последние 10–20 сообщений текущей сессии
5) новое сообщение игрока
```

**In-fiction рамка** (мягко объясняет изоляцию): «здесь, наедине» / «когда время остановилось». Не обязательно говорить прямо «это неканоническая LLM-сессия».

---

## 6. Guardrails (архитектурные запреты)

1. **Aside НЕ пишет в канон.** Никаких изменений `v2_flags`, `v2_relationships`, `v2_levels`, `v2_completed_scenes`, scenario JSON, player_state. Enforced на уровне кода (aside-слой не имеет write-доступа к этим переменным).
2. **Персонаж не знает будущего** (past-only снимок, §1).
3. **Никаких обещаний, ломающих канон.** Настоящее время о чувствах, не обещания о будущем, которое сюжет опровергнет.
4. **Память изолирована и персистентна** (§3); в канон не течёт.
5. **Safety наследуется:** платформенные фильтры + RN-SAFETY-STYLE-1 + `content_rating`/границы игрока применяются к генерации.
6. **Опционально:** сброс приватной памяти персонажа (некононично, безопасно).
7. **Save-safe:** aside-память — отдельный namespace сейва; правки/наличие aside не ломают канонические сейвы.

---

## 7. RenPy integration boundary

- Aside доступен через **opt-in вход** (как dev/test launcher N5D), **не** в основном player-facing потоке до отдельного ревью.
- Экран чата — отдельный RenPy screen; при входе игра ставится на паузу, при выходе — resume в ту же точку.
- Хуки читают `v2_*` (read-only) → снимок; **никогда** не пишут в них.
- Персона/провайдер/память вызываются через Python-tools (§2/§3/§4).

---

## 8. Failure behavior

- LLM недоступен (нет Ollama/сети) → aside-режим сообщает «недоступно», **игра не ломается**, канон нетронут.
- Невалидный/пустой снимок → aside не запускается, лог в dev-режиме.
- Ошибка записи aside-памяти → не влияет на канонический сейв.

---

## 9. Non-goals (в N6 Aside НЕ входит)

- Полный live JSON-runtime (N5G) — не требуется для Aside.
- Dev in-place edit сцены (отдельная фича, roadmap N5).
- Live-LLM, влияющий на канонические выборы/поведение (это вариант «C», ломающий детерминизм — отдельное осознанное решение, не здесь).
- Media (фото/видео) — отдельный трек.
