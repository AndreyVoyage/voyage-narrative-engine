> **Назначение:** Зафиксировать концепцию неканонической, ветвящейся, сбрасываемой «песочницы» для экспериментов с эволюцией персонажа — отдельно от канонической ДНК, сейвов Ren'Py, памяти авторской сессии PAC и обучения N8.

---

## 1. Что такое Character Evolution Sandbox

**Character Evolution Sandbox** — это неканоническая, ветвящаяся, сбрасываемая среда для экспериментов:

- автор моделирует долгосрочное изменение персонажа (эмоции, отношения, воспоминания, поведение);
- наблюдает, как персонаж эволюционирует в разных траекториях;
- сравнивает ветки эволюции;
- при желании — оформляет обоснованное **предложение на изменение канона** (proposal) для ревью владельца.

**Характер эволюции:**
- не мгновенная смена характера;
- накопление опыта, травм, привязанностей, решений;
- долгосрочные сдвиги в распределении вероятностей действий, мыслей, реакций.

Это не «автономные агенты в реальном времени» (как в `NARRATIVE_AUTONOMOUS_ENSEMBLE_CONCEPT_v1.md`) — это **авторский инструмент планирования и экспериментов с изменениями**, работающий вне игры.

## 2. Пять плоскостей (planes)

### CANON PLANE (Канон)

- `personas/`, `knowledge_base/`, `scenarios/`, `v2_*`.
- **Read-only** для PAC и Sandbox.
- Никакая автоматическая система не пишет в канон.

### AUTHORING PLANE (Авторинг)

- Промпты, варианты-кандидаты, правки человека, одобрения, состояние сессии.
- Неканоническая рабочая область PAC.
- Результат авторинга может быть:
  - перенесён в сценарий (APPROVE_SCENE);
  - сохранён в датасет (APPROVE_DATASET);
  - но **не** в канон персоны напрямую.

### EVOLUTION SANDBOX PLANE (Песочница эволюции)

- Экспериментальные снапшоты.
- Истории веток (branch histories).
- Временные воспоминания (temporary memories).
- Временные эмоциональные изменения (temporary emotional deltas).
- Временные изменения отношений (temporary relationship deltas).
- Альтернативные траектории развития.
- **Сбрасываемая (resettable).**
- **Неканоническая.**

### TRAINING DATA PLANE (Обучающие данные)

- Только примеры с `APPROVE_DATASET`.
- Кураторский отбор.
- Будущий вход для N8.
- Не модифицирует канон.

### GAME RUNTIME PLANE (Игровой рантайм)

- Активное состояние рантайма Ren'Py.
- Отдельно от авторинга, Песочницы и обучения.
- Сейвы, прогресс, флаги — в своём namespace.

## 3. Предлагаемая модель состояния Песочницы

> Все поля ниже — PROPOSED. Конкретный формат и реализация требуют разрешения D-CES-решений.

| Поле | Тип | Описание |
|------|-----|----------|
| `sandbox_session_id` | UUID v4 | Уникальный идентификатор сессии песочницы. |
| `character_id` | string | Идентификатор персонажа. |
| `parent_snapshot_id` | UUID v4 или null | Родительский снапшот (null для корня). |
| `current_snapshot_id` | UUID v4 | Текущий снапшот состояния. |
| `branch_name` | string | Человекочитаемое имя ветки. |
| `experiment_goal` | string | Описание цели эксперимента. |
| `starting_canon_snapshot` | object | Канонический снапшот на момент создания (source_commit + module hashes). |
| `non_canon_experiences` | array | Накопленный неканонический опыт. |
| `temporary_relationship_deltas` | object | Временные дельты отношений. |
| `temporary_emotional_deltas` | object | Временные эмоциональные дельты. |
| `author_notes` | string | Заметки автора. |
| `generated_scenes` | array | Сгенерированные сцены в рамках этого состояния. |
| `evaluation_results` | object | Результаты оценки ветки. |
| `reset_history` | array | История сбросов. |
| `fork_history` | array | История форков. |

### Концептуальные операции

| Операция | Описание |
|----------|----------|
| **Create from canon snapshot** | Создать песочницу из канонического снапшота персонажа. |
| **Inspect state** | Просмотреть текущее состояние песочницы. |
| **Apply non-canon experience** | Добавить неканонический опыт (сцена, событие, диалог). |
| **Generate scene** | Сгенерировать сцену с учётом текущего состояния песочницы. |
| **Compare branches** | Сравнить две ветки эволюции. |
| **Fork** | Создать ветку от текущего состояния. |
| **Reset** | Сбросить песочницу к каноническому снапшоту (или к указанному родительскому). |
| **Archive** | Архивировать песочницу (без удаления, с пометкой). |
| **Create canon-change proposal** | Создать human-reviewed proposal на изменение канона на основе наблюдений в песочнице. |

### Запрещённые операции

- ❌ **Automatic canon promotion** — автоматическое превращение состояния песочницы в канон.
- ❌ **Запись в канонические директории** (`personas/`, `knowledge_base/`, `scenarios/`, `v2_*`).
- ❌ **Влияние на игровой рантайм Ren'Py** (сейвы, прогресс).
- ❌ **Автоматическое добавление в обучающий датасет** без APPROVE_DATASET.

Любое будущее изменение канона на основе песочницы должно быть:
- явным (explicit);
- созданным или проверенным человеком;
- вне автоматического выполнения PAC/Sandbox;
- проверенным как отдельное изменение репозитория.

## 4. Различия от смежных концепций

| | Character Evolution Sandbox | Autonomous Ensemble | PAC Authoring |
|---|---|---|---|
| **Назначение** | Эксперименты с долгосрочной эволюцией персонажа. | Автономная симуляция нескольких персонажей в реальном времени. | Соавторство с моделью для написания сцен. |
| **Временной горизонт** | Дни, недели, месяцы (внутри мира). | Такты (короткие шаги). | Отдельные сцены/реплики. |
| **Канон** | Read-only. Предложения на изменение — вручную. | Non-canon by default. | Read-only. |
| **Количество персонажей** | Обычно 1 (фокус на эволюции одного). | Ансамбль (2+). | Любое. |
| **Реальное время** | Нет (офлайн-эксперименты). | Да (в будущем). | Нет. |

## 5. Границы безопасности

1. **Sandbox не пишет в канон.** Ни при каких обстоятельствах.
2. **Sandbox не влияет на игровые сейвы.** Сейвы Ren'Py и песочница — разные namespace.
3. **Sandbox не обучает модель.** Данные песочницы не попадают в N8 автоматически.
4. **Proposals проходят человеческое ревью.** Даже обоснованное предложение из песочницы — это только proposal, не commit.
5. **Сброс безопасен.** Reset возвращает состояние к каноническому снапшоту без побочных эффектов.
