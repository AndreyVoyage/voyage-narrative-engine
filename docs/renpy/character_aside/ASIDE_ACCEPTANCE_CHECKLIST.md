# Character Aside — чеклист приёмки

> Компактный, checkbox-совместимый чеклист. Применим к Character Aside и к любой
> будущей Ren'Py + LLM интеграции в этом проекте. Не заменяет
> `ASIDE_DEBUGGING_RUNBOOK.md` при реальном расследовании — это чеклист для **приёмки**
> уже (предположительно) готовой реализации.

## Static

- [ ] Ren'Py lint (`renpy.py <project> lint`) — PASS, без script errors.
- [ ] `git diff --check` — PASS, только LF/CRLF warnings допустимы.
- [ ] Worker call graph (транзитивно, не только тело функции) не содержит:
  `store`, `renpy.store`, `renpy.config`, `get_screen_variable`/`set_screen_variable`.
- [ ] Каждый `text`-displayable, показывающий динамический/недоверенный текст, имеет
  `substitute False`.
- [ ] Геометрия (composer, история) явно проверена против фактического
  `config.screen_width`/`config.screen_height` (по умолчанию 800×600, если не
  переопределено).

## Mock runtime

- [ ] Send (кнопка) отправляет сообщение и обновляет историю.
- [ ] Enter (клавиша) отправляет сообщение тем же путём, что и Send.
- [ ] Ровно один ход (`You:` + ответ) на одну отправку — нет дублей.
- [ ] История видна на экране (не только в `store`/JSONL).
- [ ] Нет необработанных исключений/traceback.

## Local runtime

- [ ] `You: <текст>` появляется немедленно после отправки.
- [ ] `[[thinking...]]`-маркер виден, пока идёт ожидание ответа.
- [ ] По завершении — либо реальный ответ, либо явная `[LLM ERROR] ...` строка (без
  тихого зависания).
- [ ] Нет утечки `[MOCK]` при `provider == "local"`.
- [ ] Настроенный таймаут (не дефолтный 30 сек, если требуется больше) реально
  применяется — проверено через быстрый CLI-смок с искусственно коротким `timeout_s`.

## UI

- [ ] Длинный ввод (>60 символов) не выходит за пределы своего `frame`.
- [ ] Message-row и action-row — раздельные, не перекрываются.
- [ ] Send/Reset/Close видны и кликабельны при заполненном поле ввода.
- [ ] Последние N записей истории (если применяется срез отображения) рендерятся
  стабильно, без "прыжков" при добавлении новой записи.
- [ ] Modal-экран не перекрыт и не заслоняет debug-overlay, оставшийся в коде по ошибке.
- [ ] Пока Aside виден, открыто стандартное подтверждение выхода (`yesno_prompt`).
- [ ] Текст quit confirmation полностью читаем, кнопки Yes/No видимы и доступны.
- [ ] Ни Aside overlay/probe, ни debug panel, ни другой custom screen не закрывает
  подтверждение выхода.
- [ ] После выбора No подтверждение закрывается, а Aside/game UI возвращается в
  нормальное состояние.

## Thread evidence (если есть JSONL/аналогичная инструментация)

- [ ] `worker_start` зафиксирован на потоке, отличном от `MainThread`.
- [ ] `provider_result`/`provider_error` зафиксированы на том же non-Main потоке.
- [ ] `main_thread_callback` зафиксирован на `MainThread`.
- [ ] `final_history_write` (или эквивалент) зафиксирован на `MainThread`.
- [ ] Оба провайдера (mock и local/аналог) протестированы в одном прогоне без сброса
  состояния между ними, чтобы подтвердить непрерывность истории.

## Git

- [ ] Staging только явными путями (`git add -- <path>`) — никогда `git add .`/`-A`.
- [ ] После каждого `git add` — проверено `git diff --cached --name-only`, что staged
  set содержит ровно ожидаемые файлы.
- [ ] Независимая QA (отдельный, скептический read-only проход) выполнена перед
  commit, не только отчёт исполнителя.
- [ ] Перед push — divergence-проверка (`git rev-list --left-right --count
  "@{u}...HEAD"`), исходящие коммиты и файлы сверены с ожиданием.
- [ ] Несвязанное dirty/untracked состояние явно перечислено и подтверждено
  нетронутым до и после операций.
