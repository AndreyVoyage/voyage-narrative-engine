# 🧠 MEMORY PROTOCOL — Voyage Narrative Engine

> **Назначение:** Сохранение, загрузка и сжатие памяти между сессиями.
> **Принцип:** Всё, что произошло — фиксируется. Всё, что устарело — сжимается.

---

## 📁 СТРУКТУРА ХРАНЕНИЯ

```
/voyage-narrative/
  /memory/
    EVENT_LOG.json          — текущие события (активная сессия)
    EVENT_LOG_ARCHIVE.json  — архив старых событий
    SUMMARY.md              — суммаризация последних 20 событий
    STATE.json              — текущее состояние персонажей и флагов
```

---

## 📝 EVENT LOG (JSON)

### Формат события

```json
{
  "id": "EV_001",
  "timestamp": "2026-06-06T11:00:00Z",
  "scene_id": "SC_001",
  "characters": ["K", "S"],
  "kira_level": "U4",
  "sergey_level": "S4",
  "flags_set": ["sergey_touched_waist"],
  "flags_unset": [],
  "emotional_anchor": "ты мой финиш",
  "summary": "Сергей положил руку на поясницу К. К не отстранилась. Взгляд в зеркале.",
  "symbol": "S1:U3→U4:K+S:gym:stretching:tension_no_release"
}
```

### Правила записи

1. **Каждая сцена = одно событие.** Если сцена длинная — одно событие на ключевой момент.
2. **Флаги обновляются сразу.** Не ждать конца сессии.
3. **Emotional anchor** — фраза, которая сработала в этой сцене (для повторного использования).
4. **Symbol** — компактный код для быстрого восстановления контекста.

## 🔄 СУММАРИЗАЦИЯ (когда Event Log > 20 событий)

### Алгоритм

1. AI читает последние 20 событий.
2. Генерирует `SUMMARY.md` (3-5 абзацев):
   - Ключевые повороты сюжета
   - Изменения уровней персонажей
   - Установленные флаги
   - Эмоциональные якоря
3. Старые 20 событий архивируются в `EVENT_LOG_ARCHIVE.json`.
4. Новый Event Log начинается с чистого листа, но с флагами и состоянием из SUMMARY.

### Пример SUMMARY.md

```markdown
# Суммаризация: Сессия SN_001 (сцены 1-20)

## Ключевые моменты
- Кира прошла путь от У1 (стойкость) до У4 (своеволие) в зале.
- Сергей перешёл от С1 (профессионал) до С4 (провокация).
- Установлены флаги: gym_stretching, sergey_touched_waist, sergey_touched_thigh.
- Кира отправила фото в топе без бюстгальтера (flag: kira_sent_photos).

## Эмоциональные якоря
- «Ты мой финиш» — использовано 3 раза, работает.
- «Собери меня» — использовано 1 раз, в пике У4.

## Запреты
- Нет запретов.

## Текущее состояние
- Кира: У4, ТГ3, дома, влажные волосы, мокрый топ.
- Сергей: С4, в зале, мокрая майка, смотрит в зеркало.
- Пользователь: приехал в зал, обнял Киру, увёз домой.
```

## 🔣 СИМВОЛИЗАЦИЯ (для быстрого восстановления)

### Формат символ-кода

```
S[id]:[K_start]→[K_end]:[chars]:[location]:[action]:[result]
```

| Поле | Описание | Пример |
|------|----------|--------|
| S[id] | ID сцены | S12 |
| [K_start]→[K_end] | Уровень Кира до→после | U3→U4 |
| [chars] | Персонажи | K+S (Кира + Сергей), K+S+Y (втроём) |
| [location] | Локация | gym, bar, hotel |
| [action] | Действие | stretching, flirting, kissing |
| [result] | Результат | tension_no_release, kiss, aftercare |

### Примеры

```
S12:U3→U4:K+S:gym:stretching:tension_no_release
S13:U4→U4:K+S:gym:touching:sergey_withdraws
S14:U4→U7:K+Y:gym:embrace:user_rescues
S15:U7→U7:K+Y:home:night:aftercare_sleep
```

## 💾 СОХРАНЕНИЕ И ЗАГРУЗКА

### Сохранение (в конце сессии)

```bash
# 1. Обновить STATE.json
# 2. Дописать в EVENT_LOG.json
# 3. Если Event Log > 20 — запустить суммаризацию
# 4. Сохранить SUMMARY.md
# 5. Git commit (если репозиторий)
```

### Загрузка (в начале новой сессии)

```bash
# 1. Прочитать STATE.json
# 2. Прочитать SUMMARY.md (если есть)
# 3. Прочитать последние 5 событий из EVENT_LOG.json
# 4. Собрать PROMPT.txt = CORE + PERSONAS + STATE + LAST_EVENTS
```

### Локальное хранение (без репозитория)

    SUMMARY.md

## 🧹 ОЧИСТКА И АРХИВАЦИЯ

### Когда архивировать?

- Event Log > 20 событий
- Сессия длится > 3 часов
- Пользователь явно просит: «суммаризируй»

### Что архивировать?

- Все события кроме последних 5
- Старые SUMMARY.md (переименовать в SUMMARY_001.md)

### Что НЕ архивировать?

- STATE.json (всегда актуален)
- Флаги (всегда актуальны)
- Emotional anchors (всегда актуальны)
