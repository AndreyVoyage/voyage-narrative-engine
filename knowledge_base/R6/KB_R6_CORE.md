## 1. НАЗНАЧЕНИЕ R6

**Роль:** Сборщик модулей. Агрегирует данные из R1 (портрет), R2 (психология), R3 (сексология), R4 (речь), R5 (визуал) в единый Persona Module.

**Критично:**
- Не изменять данные предыдущих ролей — только агрегировать.
- Создавать PREVIOUS / CURRENT / FUTURE состояния для каждого подуровня.
- Создавать HUMAN / AUTONOMOUS / META контейнеры.
- Создавать отношения (сцена + тон).

**Не делает:**
- Не переписывает ФМДР (это R4).
- Не меняет ВСЦНО (это R2).
- Не создаёт новый визуал (это R5).

## 2. СТРУКТУРА PERSONA MODULE

### 2.1 Корневые файлы (3 файла)

| Файл | Назначение | Формат |
|------|-----------|--------|
| **INDEX.json** | Метаданные, версия, иерархия, словари, порядок загрузки | JSON |
| **ASSEMBLY.md** | Карта сборки, workflow, цепочка зависимостей | Markdown |
| **HANDOFF_R6.md** | Финальная компрессия и передача в рантайм | Markdown |

### 2.2 HUMAN контейнер (7 папок)

```
core/          — Портрет (R1), ВСЦНО (R2), Identity (R1)
levels/U1-A…U7-B/ — 14 папок, каждая с:
  - FUTURE.md (3 примера ФМДР + текущий визуал)
  - CURRENT.md (2 примера ФМДР + текущий визуал)
  - PREVIOUS.md (1 пример ФМДР + предыдущий визуал)
psychology/    — Травма, защиты, триггеры, речевые паттерны (R2)
sexology/      — TEC, сексуальные сценарии, предыдущий опыт (R3)
visual/        — Anatomic Anchor, Visual Signature, Dynamic Visuals (R5)
dynamics/      — Типы движения, signature items, агрессия/нежность (R5)
memory/        — Якоря, триггеры, ассоциативные сети (R1+R2)
relationships/ — С кем, сцена, тон (R1+R6)
environment/   — Бар, квартира, работа, флора/фауна (R1)
safety/        — Мониторинг ВРИ, сейф-коды, экскалация (R2+R3)
autonomous/    — Пустая зарезервированная папка для автономного поведения
meta/          — Метаданные, ченджлог, тест-кейсы
```

### 2.3 AUTONOMOUS контейнер (зарезервирован)

## 3. ФОРМАТ RELATIONSHIPS (Отношения)

**Формат:** JSON для каждого персонажа:

```json
{
  "with": "Elena",
  "scene": "corporate_office",
  "tone": "formal_hidden_tension",
  "history": "2 years colleagues, one night drunk",
  "current_status": "avoidant_U3-B",
  "power_dynamic": "equal_formal",
  "triggers": ["she_looks_at_phone", "he_adjusts_watch"]
}
```

**Правила:**
- Сцена: одно слово, snake_case (bar, office, bedroom, street).
- Тон: 2–3 токена, snake_case (formal_hidden_tension, playful_avoidant).
- История: 1–2 предложения.
- Статус: текущий подуровень (U3-B = избегающий, отстранённый).

---

## 4. ФОРМАТ PREVIOUS / CURRENT / FUTURE

### PREVIOUS.md (1 пример)
- ФМДР-пример из предыдущего подуровня.
- Визуал из предыдущего подуровня (для контекста «откуда пришёл»).

### CURRENT.md (2 примера)
- ФМДР-примеры текущего подуровня (2 штуки — для вариативности).
- Визуал текущего подуровня.

### FUTURE.md (3 примера)
- ФМДР-примеры следующего подуровня (3 штуки — для направления).
- Визуал следующего подуровня (для «куда идёт»).

**Правило:** Не писать «он будет бояться» — только конкретные мысли/действия/речи.

---

## 5. ИНТЕГРАЦИЯ С КОМПРЕССИЕЙ (R7)

R6 создаёт структуру, R7 (Refactor) сжимает:
- 12 папок → 1 монофайл (runtime).
- 3 слоя контекста: BASE + STATE + LIVE.
- COMPACT-версии для каждого подуровня.

**Правило:** R6 не сжимает — только создаёт структуру для сжатия.
