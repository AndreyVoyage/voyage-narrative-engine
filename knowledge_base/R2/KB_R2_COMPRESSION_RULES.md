## ПРАВИЛО КОМПРЕССИИ

- **Что сжимается:** матрицы, таблицы, повторяющиеся термины, избыточные примеры.
- **Что НЕ сжимается:** safety-протоколы, stop-words, emergency procedures, TEC-рекомендации для R3.
- **Цель:** уменьшить токен-затраты на ~55-65% при сохранении 100% функциональности для R3.
- **Формат выхода:** Markdown с компрессорными метками (`_linear_table`, `_base_delta`, `_glossary`).

## МЕТОД 1: `_linear_table` (VSCNO + AD)

**Применение:** VSCNO-матрица 14×4 (14 подуровней × 4 оси) и AD-матрица 14×4 (14 подуровней × доступные/запрещённые коды).
**Экономия:** ~60% токенов.
**Критичность:** ОБЯЗАТЕЛЬНА для R2.

### Правило преобразования

ИЗ (развёрнутый формат, ~200 строк):
```markdown
### У1-А
- ВЛ: 1
- СТ: 4
- НЖ: 2
- ОГ: 3
- Доступные АД: ПД, ПУ
- Запрещённые АД: ФС, ЛС, ПР

### У1-Б
- ВЛ: 2
- СТ: 3
- НЖ: 2
- ОГ: 3
- Доступные АД: ПД, ПУ, КН
- Запрещённые АД: ФС, ЛС
...
```

В (COMPACT, ~30 строк):
```markdown
<!-- _linear_table: VSCNO_BASELINE -->
| У | ВЛ | СТ | НЖ | ОГ | Доступные АД | Запрещённые АД |
|---|----|----|----|----|--------------|----------------|
| 1А | 1 | 4 | 2 | 3 | ПД, ПУ | ФС, ЛС, ПР |
| 1Б | 2 | 3 | 2 | 3 | ПД, ПУ, КН | ФС, ЛС |
| 2А | 2 | 4 | 2 | 2 | ПД, ПУ, КН, ФС* | ЛС, ПР |
| 2Б | 3 | 3 | 2 | 2 | КН, ФС, ЛС | ПР, СЛ |
| 3А | 2 | 3 | 3 | 2 | ПУ, ПД, СП | ПР, ФС |
| 3Б | 3 | 2 | 3 | 2 | КН, ФС, ЛС, ПР | ПУ, ПД |
| 4А | 4 | 1 | 2 | 3 | ФС, ЛС, ПР, СЛ | ПУ, ПД |
| 4Б | 4 | 1 | 2 | 3 | Все кроме ПУ, ПД | ПУ, ПД |
| 5А | 3 | 2 | 3 | 2 | ПУ, ФС, СП, КН | СЛ, ПР |
| 5Б | 4 | 1 | 3 | 2 | ФС, ЛС, КН, ВС | СЛ |
| 6А | 2 | 2 | 3 | 3 | СП, СЛ, ПР | ФС, ЛС |
| 6Б | 2 | 1 | 4 | 3 | ВС, ДР | Все остальные |
| 7А | 2 | 1 | 3 | 4 | СЛ, ПР, ФС | ПУ, ПД, ВС |
| 7Б | 3 | 1 | 3 | 3 | СЛ, ПР, ФС | ПУ, ПД, ВС, СП |
<!-- _end_linear_table -->

*ФС — доступен случайно, с низкой вероятностью.
```

### Проверка после компрессии
```
□ Сумма ВЛ+СТ+НЖ+ОГ = 10 для каждой строки.
□ 14 строк присутствуют (1А…7Б).
□ AD-коды соответствуют канону из AD_AVAILABILITY_MATRIX.md.
□ Все [ADAPTED] отмечены звёздочкой (*) или комментарием.
```

## МЕТОД 2: `_base_delta` (Internal State)

**Применение:** Internal State (desire, anxiety, tension, frustration) по 14 подуровням.
**Экономия:** ~50% токенов.
**Критичность:** ОБЯЗАТЕЛЬНА для R2.

### Правило преобразования

ИЗ (развёрнутый, ~150 строк):
```markdown
### У1-А
- desire: 2
- anxiety: 8
- desire_tension: 7
- frustration: 5

### У1-Б
- desire: 3
- anxiety: 7
- desire_tension: 6
- frustration: 4
...
```

В (COMPACT, ~40 строк):
```markdown
<!-- _base_delta: INTERNAL_STATE -->
**Базовый профиль (У1-А):** desire=2, anxiety=8, tension=7, frustration=5

| У | Δ desire | Δ anxiety | Δ tension | Δ frustration | Итого |
|---|----------|-----------|-----------|---------------|-------|
| 1А | 0 | 0 | 0 | 0 | 2,8,7,5 |
| 1Б | +1 | -1 | -1 | -1 | 3,7,6,4 |
| 2А | +1 | -2 | -2 | -1 | 3,6,5,4 |
| 2Б | +2 | -3 | -2 | -1 | 4,5,5,4 |
| 3А | +2 | -2 | -1 | 0 | 4,6,6,5 |
| 3Б | +3 | -3 | -2 | -1 | 5,5,5,4 |
| 4А | +4 | -5 | -3 | -2 | 6,3,4,3 |
| 4Б | +5 | -5 | -2 | -2 | 7,3,5,3 |
| 5А | +3 | -3 | -1 | -1 | 5,5,6,4 |
| 5Б | +4 | -4 | -2 | -1 | 6,4,5,4 |
| 6А | +2 | -4 | -2 | -2 | 4,4,5,3 |
| 6Б | +1 | -5 | -3 | -3 | 3,3,4,2 |
| 7А | +2 | -6 | -4 | -3 | 4,2,3,2 |
| 7Б | +3 | -6 | -4 | -3 | 5,2,3,2 |
<!-- _end_base_delta -->

**Проверка:** Итоговые значения должны быть ∈ [0,10]. Если нет — ошибка в дельтах.
```

### Проверка после компрессии
```
□ Базовый профиль (У1-А) соответствует исходному.
□ Все итоговые значения ∈ [0,10].
□ Дельты логичны: при переходе вверх desire растёт, anxiety падает (в общем случае).
```

## МЕТОД 3: `_glossary` (Психологические термины)

**Применение:** Повторяющиеся психологические концепции (стили привязанности, Big Five, IFS, травмы, когнитивные искажения).
**Экономия:** ~40% токенов.
**Критичность:** РЕКОМЕНДУЕТСЯ для R2.

### Правило преобразования

ИЗ (развёрнутый, с повторениями):
```markdown
Стиль привязанности: anxious-preoccupied (Ainsworth/Main, характеризуется страхом отвержения, гиперактивацией привязанности, эмоциональной зависимостью...).

Когнитивное искажение: катастрофизация (Beck, преувеличение негативных последствий...).
```

В (COMPACT, с глоссарием):
```markdown
<!-- _glossary: R2_PSYCHOLOGY -->
**Коды стилей привязанности:**
- AP = anxious-preoccupied (Ainsworth/Main)
- AV = avoidant-dismissive
- SE = secure
- DI = disorganized

**Коды когнитивных искажений:**
- CAT = катастрофизация (Beck)
- MIN = минимизация позитивного
- MIND = ментальное чтение
- ALL = чёрно-белое мышление

**Коды IFS-частей:**
- M = Manager (защитная)
- F = Firefighter (импульсивная)
- E = Exile (травматическая)
- S = Self (центральная)
<!-- _end_glossary -->

Стиль привязанности: **AP** (характеризуется страхом отвержения, гиперактивацией привязанности, эмоциональной зависимостью).

Когнитивное искажение: **CAT** (преувеличение негативных последствий).
```

### Проверка после компрессии
```
□ Глоссарий содержит все уникальные термины, используемые в тексте.
□ Первое упоминание термина — с полным описанием + код.
□ Последующие упоминания — только код.
□ Коды не двусмысленны (AP ≠ AV, CAT ≠ MIN).
```

## МЕТОД 4: `_fmrd_fold` (НЕ применяется к R2)

**Правило:** `_fmrd_fold` используется только для R4 (Linguist), где >15 примеров ФМДР сокращаются до 3 ключевых.
**В R2:** мысли по 14 подуровням — это не ФМДР-примеры, а психологический внутренний диалог. Они НЕ сокращаются до 3, но могут быть сжаты через `_glossary` и `_base_delta`.

---

## МЕТОД 5: `_tec_fold` (НЕ применяется к R2)

**Правило:** `_tec_fold` используется только для R3 (Sexologist), где 8 TEC JSON сокращаются до компактных блоков.
**В R2:** TEC-рекомендации — это 8 текстовых пунктов, не JSON. Они остаются полными, но могут быть сжаты через `_glossary`.

---

## МЕТОД 6: `_relation_fold` (НЕ применяется к R2)

**Правило:** `_relation_fold` используется для R6 (Modular Architect), где таблица отношений 8×4 сворачивается в строку.
**В R2:** реляционная сетка — это матрица, но она не 8×4, а N×M (N = персонажи, M = параметры). Сжатие через `_linear_table` или оставление полной (если N ≤ 6).

## ПРОВЕРКА КОМПРЕССИИ (POST-COMPRESSION CHECK)

```
□ COMPACT-файл содержит ВСЕ обязательные секции (1–8).
□ Секция 2 (VSCNO) читаема без полного файла (самодостаточна).
□ Секция 3 (Internal State) читаема без полного файла.
□ Секция 6 (TEC-рекомендации) — полные, не сокращены до непонятности.
□ Секция 7 (Safety) — полная, не сжата.
□ Размер COMPACT < 50% от полного файла (целевой показатель).
□ Если размер > 50% → проверить, не забыли ли применить компрессор.
```
