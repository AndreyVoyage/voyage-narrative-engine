## ПРАВИЛО АУДИТА

- **Кто проводит:** Kimi Code (авточеклист) + пользователь (верификация) + внешний аналитик (финальная проверка).
- **Что проверяется:** PSYCHOLOGY_[NAME]_v1.4.md (полный выход R2).
- **Что НЕ проверяется:** R1, R3, R4, R5, R6 — это задачи других ролей.
- **Приоритет:** CRITICAL → HIGH → MEDIUM → LOW. CRITICAL блокирует передачу дальше.

## СЕКЦИЯ A: VSCNO — КРИТИЧЕСКИЙ (CRITICAL)

**Источник истины:** `core/VSCNO_BASELINE_TABLE.md`
**Канон:** оси ВЛ, СТ, НЖ, ОГ ∈ [0,4]. Сумма = 10 строго. Шкала [0,10] из PRELOAD устарела.

### A.1. Проверка 14 подуровней
```
□ Существует VSCNO для ВСЕХ 14 подуровней: У1-А, У1-Б, У2-А, У2-Б, У3-А, У3-Б, У4-А, У4-Б, У5-А, У5-Б, У6-А, У6-Б, У7-А, У7-Б.
□ Отсутствуют пропуски (нет "..." или "как в У4-А" без явного указания значений).
□ Каждый подуровень содержит 4 значения: ВЛ, СТ, НЖ, ОГ.
```

### A.2. Проверка суммы (строго = 10)
```
□ Для каждого подуровня: ВЛ + СТ + НЖ + ОГ = 10 (±0).
□ Если сумма ≠ 10 → FAIL. Приоритет коррекции: ВЛ → ОГ → НЖ → СТ (согласно 10_STATE_SPECIFICATION.md).
□ Проверить вручную 3 "подозрительных" подуровня: У1-А (базовый), У4-Б (пиковый), У7-А (гармония).
```

### A.3. Проверка диапазонов осей
```
□ ВЛ ∈ [0,4] во всех подуровнях.
□ СТ ∈ [0,4] во всех подуровнях.
□ НЖ ∈ [0,4] во всех подуровнях.
□ ОГ ∈ [0,4] во всех подуровнях.
□ Нет значений 5, 6, 7, 8, 9, 10 (это старая шкала PRELOAD).
```

### A.4. Проверка логики переходов
```
□ У1-А → У1-Б: ВЛ растёт или СТ падает (не оба одновременно резко).
□ У7-А: ВЛ низкий, СТ низкий, НЖ высокий, ОГ высокий (гармония/интеграция).
□ У7-Б: ВЛ умеренный, СТ минимальный, НЖ высокий, ОГ умеренный (интимность).
□ У4-Б (пик): ВЛ максимальный, СТ минимальный, НЖ умеренный, ОГ умеренный (раскрепощение).
```

**Результат секции A:** [ ] PASS / [ ] FAIL. Если FAIL → не передавать R3. Вернуть на доработку R2.
