## СЕКЦИИ АУДИТА R4

### A. ФОРМАТ ФМДР — КРИТИЧЕСКИЙ

```
□ Мысль: 1-2 предложения (не 3+, не абзац).
□ Мысль: не психологический анализ (короткая связь с R2).
□ Действие: визуально наблюдаемое (нет внутренних состояний).
□ Речь: в кавычках-ёлочках «».
□ Речь: нет табу-слов (проверить по списку).
□ Если без слов — явно указать [нет слов, только ...].
```

### B. SPEECH PROFILE MATRIX

```
□ 14 строк (У1-А … У7-Б).
□ 6 столбцов заполнены (тон, темп, словарь, thought_length, детальность, фраза).
□ Thought_length логичен: У1-А = 1 предл., У5-А = 1 слово, У7-А = 2-3 предл.
□ Детальность: У1-А = минимальная, У5-А = максимальная, У7-А = средняя.
```

### C. CODE-SWITCHING

```
□ 4 контекста описаны: работа, друзья, интим, стресс.
□ Каждый контекст имеет свой тон, темп, лексику.
□ Контекст «стресс» = фасад/защита (не открытость).
□ Контекст «интим» = уязвимость/честность (не фасад).
```

### D. УНИКАЛЬНОСТЬ

```
□ Фраза (signature) не похожа на других персонажей (проверить по базе).
□ Словарь уникален (не generic «привет, как дела»).
□ Тон уникален (не все персонажи «игривые»).
□ Code-switching различается между персонажами.
```

### E. СВЯЗЬ С R2/R3

```
□ Речь отражает стиль привязанности R2 (anxious = забота, avoidant = отстранение).
□ Речь в интимном контексте = TEC-стиль R3 (SPON/RESP/MIX).
□ Нет противоречий: если R2 = anxious, речь не может быть cold/predatory без объяснения.
```

### F. ФОРМАТ ФАЙЛА

```
□ LINGUISTICS_[NAME]_v1.4.md — Markdown.
□ Speech Profile Matrix — таблица 14×6.
□ ФМДР-примеры: минимум 3 (У2-А, У4-А, У7-А).
□ Code-Switching: 4 контекста.
□ COMPACT-версия: _fmrd_fold (если >15 примеров).
```
